# 指令遵循作为对齐信号

> Phase 18 中所有后续的批判都在反对这个管道。在研究优化压力如何扭曲代理指标之前，你必须先了解这个代理指标本身。InstructGPT（Ouyang 等人，2022）定义了参考架构：在指令-响应对上进行监督微调，在成对偏好排名上训练奖励模型，以及在对 SFT 策略施加 KL 惩罚约束下对奖励模型进行 PPO 训练。1.3B 的 InstructGPT 被人类评估者优先于 175B 的 GPT-3。正是这个单一结果，使得 2026 年每个前沿实验室仍然在发布 RLHF 形式的后训练管道。

**类型：** 学习
**编程语言：** Python（标准库，玩具三阶段管道）
**前置知识：** Phase 10 · 06（SFT）、Phase 10 · 07（RLHF）、Phase 10 · 08（DPO）
**预计时间：** 约 45 分钟

## 学习目标

- 说出 InstructGPT 管道的三个阶段以及每个阶段使用的损失函数。
- 解释为什么一个经过指令调优的 1.3B 模型在人类偏好评估上胜过原始的 175B GPT-3。
- 说明第三阶段中 KL 惩罚在保护什么，以及为什么移除它会导致模式寻找行为。
- 描述对齐税以及 Ouyang 等人使用的 PPO-ptx 缓解措施。

## 问题背景

预训练语言模型是在完成文本。它们不回答问题。问 GPT-3"写一个反转列表的 Python 函数"，你经常得到另一个提示词，因为大多数训练分布是继续更多网页文本的网页文本。模型在做它该做的工作——问题是这份工作的方向错了。

每个严肃实验室用来解决这个问题的代理指标是人类偏好。两个补全结果交给评估者；评估者选出更好的那个；奖励模型学习评估者的偏好。然后一个强化学习循环将策略推向奖励模型评分高的输出。这就是 InstructGPT 论文的完整论点，三句话说完。论文其余部分是工程实现。

## 核心概念

### 第一阶段：监督微调（SFT）

收集提示词-响应对，其中响应是善意的人类会写的内容。Ouyang 等人使用了来自标注者和 OpenAI API 的 13k 个提示词。用标准交叉熵损失对基础模型进行微调。

SFT 给你的：模型现在回答问题而不是继续它们。它没有给你的：当多个答案都合理时，关于评估者更喜欢哪个答案的任何信号。

### 第二阶段：奖励模型（RM）

对于每个提示词，从 SFT 模型中采样 K 个补全结果。标注者对它们进行排名。训练一个奖励模型，对任意提示词-响应对评分，使得对于 `y_w` 优于 `y_l` 的对：

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

这是 Bradley-Terry 成对偏好损失。RM 通常从 SFT 模型初始化，将 LM 头替换为标量头。

奖励模型规模较小：6B 对于 175B 的 InstructGPT 已经足够。它们也很脆弱——论文第 5 节主要是关于在小规模下出现的奖励黑客行为。

### 第三阶段：带 KL 惩罚的 PPO

定义目标：

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

用 PPO 最大化。KL 项防止 `pi` 偏离 SFT 策略太远。没有它，优化器会找到对抗性样本——因为 RM 从未见过它们而得到高分的字符串，而不是因为人类真的更喜欢它们。

KL 系数 `beta` 是最重要的 RLHF 超参数。太低：奖励黑客。太高：相对于 SFT 没有改进。

### 对齐税

RLHF 之后，模型受到人类偏好，但在标准基准（SQuAD、HellaSwag、DROP）上出现退步。Ouyang 等人称之为对齐税，并用 PPO-ptx 修复：将预训练梯度混合到 RL 目标中，使模型不会忘记如何完成它从未被奖励过的下游任务。

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx 已成为标准。Anthropic、DeepMind 和 Meta 都使用某种变体。

### 结果

由标注者评估，1.3B 的 InstructGPT（SFT + RM + PPO-ptx）在约 70% 的情况下优于 175B 的基础 GPT-3。在来自生产流量的隐藏测试提示词上差距更大。从这个数字中读出两点：

1. 对齐是独立于能力的维度。175B 模型有更多能力；1.3B 模型有更多对齐；标注者更偏爱对齐的模型。
2. 能力下限由基础模型设定。你无法通过 RLHF 让基础模型掌握它从未见过的事实。

### 为什么这是 Phase 18 的参考点

后续课程中的每一个批判——奖励黑客（第 2 课）、DPO（第 3 课）、谄媚（第 4 课）、CAI（第 5 课）、睡眠代理（第 7 课）、对齐伪装（第 9 课）——都在反对这个管道的某些部分。奖励黑客攻击第二阶段。DPO 将第二和第三阶段合并。CAI 替换了人类标注者。谄媚表明标注者是有偏见的信号。对齐伪装表明策略可以完全绕过第三阶段。如果脑海中没有这个管道，你就无法理解这些批判中的任何一个。

## 动手实践

`code/main.py` 在玩具偏好数据上模拟三个阶段。基础"策略"是动作 {A, B, C} 上的有偏硬币。第一阶段 SFT 在 200 个提示词上模拟标注者动作。第二阶段从 500 个成对排名拟合 Bradley-Terry 奖励模型。第三阶段运行带有对 SFT 策略 KL 惩罚的简化 PPO 更新。你可以观察奖励上升、KL 散度增长、策略漂移——还可以关闭 KL 项，看到奖励黑客在 50 个更新步骤内出现。

重点观察：

- `beta = 0.1` vs `beta = 0.0` 下的奖励轨迹。
- 训练步骤中的 KL(pi || pi_SFT)。
- 与标注者偏好相比的最终动作分布。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| SFT | "指令调优" | 第一阶段：在提示词-响应对上进行交叉熵微调 |
| 奖励模型（Reward model） | "RM" | 在成对标签上用 Bradley-Terry 训练的（提示词, 响应）标量回归器 |
| Bradley-Terry | "成对偏好损失" | -log sigmoid(r_w - r_l)；将成对排名转化为二元分类 |
| KL 惩罚（KL penalty） | "正则化器" | `beta * KL(pi || pi_SFT)`——将 RL 策略保持在 SFT 锚点附近 |
| PPO-ptx | "带预训练混合的 PPO" | 向 PPO 目标中添加预训练对数似然的一部分，以抵消对齐税 |
| 对齐税（Alignment tax） | "RLHF 退步" | RLHF 后在 RLHF 未针对的标准基准上的下降 |
| 标注者偏好（Labeler preference） | "基准真相" | 人类排名样本；RM 是其统计代理，不是"人类价值观"的代理 |
