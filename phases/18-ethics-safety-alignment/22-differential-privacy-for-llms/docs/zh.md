# 大语言模型的差分隐私

> DP-SGD 仍是标准方案——注入噪声的梯度更新提供形式化的 (epsilon, delta) 保证。在计算、内存和效用方面的开销是显著的；参数高效的差分隐私微调（LoRA + DP-SGD）是 2025 年的常见配置（ACM 2025）。两条证据线相互矛盾：基于金丝雀的成员推断（Duan 等人，2024）报告对语言模型的成功率有限；训练数据提取（Carlini 等人，2021；Nasr 等人，2025）恢复了大量逐字记忆内容。解决方案（arXiv:2503.06808，2025 年 3 月）：差距在于测量对象——插入的金丝雀 vs"最可提取"数据。新的金丝雀设计无需影子模型即可进行基于损失的成员推断攻击，并产生了首次对真实数据上具有现实 DP 保证的大语言模型的非平凡 DP 审计。替代方案：PMixED（arXiv:2403.15638）——通过下一词分布的专家混合在推理时实现私密预测；差分隐私合成数据生成（Google Research 2024）。新兴攻击：通过大语言模型反馈的差分隐私逆转——置信度分数泄露。

## 问题背景

大语言模型会记忆内容。Carlini 等人 2021 年的研究表明，生产语言模型能按需复现训练文本中的逐字内容。差分隐私（Differential Privacy，DP）是形式化防御手段：通过训练使输出在形式上对任意单个训练样本不敏感。2024-2025 年的证据表明 DP-SGD 是必要的，但部署的 ε 值可能与威胁模型不匹配。

## 核心概念

### (ε, δ)-差分隐私

若随机算法 M 对于任意两个仅差一个样本的数据集以及任意事件 S 满足：
P(M(D) in S) <= e^ε * P(M(D') in S) + δ，
则 M 满足 (ε, δ)-DP。

解读：输出分布足够接近（由 ε 参数化），任何单个个体的贡献都无法被可靠推断，除非概率超过 δ。

### DP-SGD

Abadi 等人 2016 年。标准方案：
1. 采样一个小批次。
2. 计算每样本梯度。
3. 将每个样本的梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和并加入标准差为 σ * C 的高斯噪声。
5. 用含噪之和更新参数。

隐私代价由计量器跟踪（矩计量器、Rényi DP 计量器）。大语言模型文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而差异很大；没有普遍"安全"的默认 ε 值。已发表的示例中，某些大语言模型训练场景的 ε 约在 1-10 之间，但这些仅为说明性数值，而非推荐默认值。较低的 ε 通常需要更多噪声，可能增大效用损失。

### LoRA + DP-SGD

对前沿模型进行完整的 DP-SGD 训练代价过高。LoRA（Hu 等人 2022）将梯度更新限制在小型适配器上，减少了每样本梯度的存储需求。LoRA + DP-SGD 是 2025 年的常见配置。DP 保证适用于适配器；基础模型保持固定。

### 2024-2025 年的矛盾

两条证据线：

- **金丝雀成员推断攻击（Duan 等人 2024）。** 向训练数据中插入唯一金丝雀，测量成员推断攻击者能否识别它们。报告对语言模型的成功率有限，表明成员推断攻击（MIA）是困难的。
- **训练数据提取（Carlini 2021，Nasr 等人 2025）。** 用前缀提示模型，测量其是否能恢复训练中的逐字文本。报告了大量记忆内容，表明 MIA 在相关意义上是容易的。

2025 年 3 月的解决方案（arXiv:2503.06808）：两者测量的是不同事物。MIA 询问"样本 e 是否在 D 中？"针对插入的金丝雀。提取询问"我能恢复 D 的什么内容？""最可提取"的样本才是隐私的关键；金丝雀低估了这一点，因为金丝雀并未被优化为可提取的。

新的金丝雀设计：无需影子模型的基于损失的 MIA。首次对真实数据上具有现实 DP 保证的大语言模型的非平凡 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时私密预测。下一词分布上的专家混合；每个专家看到训练数据的一个分片；聚合时添加噪声以实现 DP。完全避免了 DP 训练。
- **差分隐私合成数据生成（Google Research 2024）。** 用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两者都规避了完整 DP 训练的效用代价，但代价是不同的威胁模型。

### 通过大语言模型反馈的差分隐私逆转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言机来重新识别个体。即使输出不泄露，置信度分布也可能泄露。

防御措施：不暴露置信度，或在暴露前截断/量化置信度。这是超出 (ε, δ)-DP 训练之外的额外要求。

## 动手实践

`code/main.py` 在玩具二元分类数据集上模拟 DP-SGD。你可以调整噪声倍增因子 σ 和裁剪范数 C，追踪 (ε, δ) 预算和准确性代价。"金丝雀攻击"插入一个唯一训练样本，测量差分隐私前后基于对数损失的测试能否检测到它。

## 产出技能

本课产出 `outputs/skill-dp-audit.md`。给定语言模型部署中的 DP 声明，对以下内容进行审计：(ε, δ) 值、所用计量器、MIA 评估协议，以及置信度暴露向量是否已被评估。

1. 运行 `code/main.py`。在 σ ∈ {0.5, 1.0, 2.0} 上扫描并报告 (ε, δ)-准确性权衡。找出效用崩溃的点。

2. 实现金丝雀插入和基于对数损失的测试。测量在 σ = 1.0 下 DP-SGD 前后的检测率。

4. 设计一个完全在推理时运行的 PMixED（arXiv:2403.15638）部署。PMixED 解决了哪些 DP-SGD 未能解决的威胁模型？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| DP | "(ε, δ)-差分隐私" | 形式化隐私：相邻数据集下输出分布足够接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效私密微调" | 在低秩适配器上进行 DP-SGD；2025 年标准配置 |
| MIA | "成员推断攻击" | 判断样本是否在训练数据中的攻击 |
| Canary（金丝雀） | "插入水印样本" | 用于测量 DP 泄露的唯一训练样本 |
| PMixED | "私密推断混合" | 通过下一词分布专家混合实现推理时 DP |
| DP Reversal（DP 逆转） | "置信度泄露攻击" | 利用模型置信度作为预言机进行重新识别的攻击 |
