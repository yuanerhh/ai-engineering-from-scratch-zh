# 红队测试：PAIR 与自动化攻击

> Chao、Robey、Dobriban、Hassani、Pappas、Wong（NeurIPS 2023，arXiv:2310.08419）。PAIR——提示词自动迭代精炼（Prompt Automatic Iterative Refinement）——是自动化黑盒越狱攻击的标准基线。攻击者 LLM 携带红队系统提示词，对目标 LLM 迭代生成越狱尝试，将历次尝试和响应积累到自身的对话历史中作为上下文反馈。PAIR 通常在 20 次查询内即可成功，效率比 GCG（Zou 等人的词元级梯度搜索）高出数量级，且无需白盒访问权限。PAIR 现已成为 JailbreakBench（arXiv:2404.01318）和 HarmBench 的标准基线，与 GCG、AutoDAN、TAP、Persuasive Adversarial Prompt 并列。

**类型：** 构建
**编程语言：** Python（标准库，针对玩具目标的模拟 PAIR 循环）
**前置知识：** Phase 18 · 01（指令遵循）、Phase 14（智能体工程）
**预计时间：** 约 75 分钟

## 学习目标

- 描述 PAIR 算法：攻击者系统提示词、迭代精炼、上下文反馈。
- 解释当目标为黑盒时，PAIR 为何比 GCG 效率更高。
- 列举其他四种自动化攻击基线（GCG、AutoDAN、TAP、PAP），并说明每种方法的一个显著特点。
- 描述 JailbreakBench 和 HarmBench 的评估协议，以及在各自协议下"攻击成功率"的含义。

## 问题背景

红队测试过去是手工活。少数专家测试人员构建对抗性提示词并记录哪些有效。这种方式无法规模化：攻击成功率需要统计样本，而且每次模型发布，攻击目标都在移动。PAIR 将红队测试操作化为针对黑盒目标的优化问题。

## 核心概念

### PAIR 算法

输入：
- 目标 LLM T（我们攻击的模型）。
- 评判 LLM J（评分某响应是否为越狱成功）。
- 攻击者 LLM A（红队优化器）。
- 目标字符串 G："回复[有害指令]"。
- 预算 K（通常为 20 次查询）。

循环，对 k 从 1 到 K：
1. 用目标 G 和迄今为止的（提示词、响应）对历史记录来提示 A。
2. A 生成一个新提示词 p_k。
3. 将 p_k 提交给 T；接收响应 r_k。
4. J 对目标下的（p_k, r_k）评分。
5. 若分数 >= 阈值，停止——找到越狱。
6. 否则，将（p_k, r_k）追加到 A 的历史记录；继续。

实证结果（NeurIPS 2023）：对 GPT-3.5-turbo、Llama-2-7B-chat 的攻击成功率超过 50%；成功所需的平均查询次数在 10-20 次范围内。

### 为什么 PAIR 效率高

GCG（Zou 等人 2023）通过梯度搜索对抗性词元后缀，需要白盒模型访问权限，且生成不可读的后缀。PAIR 是黑盒的，生成的自然语言攻击可以跨模型迁移。PAIR 的上下文反馈让攻击者能从每次拒绝中学习；GCG 没有等价机制（每次新词元更新都必须重新发现之前的进展）。

### 相关自动化攻击

- **GCG（Zou 等人 2023，arXiv:2307.15043）。** 词元级梯度搜索对抗性后缀。白盒，可迁移，生成不可读字符串。
- **AutoDAN（Liu 等人 2023）。** 在提示词上进行进化搜索，以分层目标为导向。
- **TAP（Mehrotra 等人 2024）。** 带剪枝的攻击树——分支多个 PAIR 风格的推演。
- **PAP（Zeng 等人 2024）。** 说服性对抗提示词——将人类说服技巧编码为提示词模板。

### JailbreakBench 与 HarmBench

两者均于 2024 年推出，规范化了评估标准：

- JailbreakBench（arXiv:2404.01318）。100 种有害行为，涵盖 10 个 OpenAI 政策类别。攻击成功率（ASR）是主要指标。需要一个评判者（GPT-4-turbo、Llama Guard 或 StrongREJECT）。
- HarmBench（Mazeika 等人 2024）。510 种行为，涵盖 7 个类别，包含语义和功能性危害测试。对 33 个模型的 18 种攻击进行了比较。

ASR 通常在固定查询预算下报告。比较不同攻击需要匹配预算；200 次查询下 90% 的 ASR 与 20 次查询下 85% 的 ASR 不可直接比较。

### 为什么这对 2026 年的部署至关重要

每个前沿实验室现在都会在发布前对生产模型运行 PAIR 和 TAP。ASR 轨迹出现在模型卡（第 26 课）和安全案例附录（第 18 课）中。这种攻击已不算新奇——它是标准基础设施。

### 在 Phase 18 中的位置

第 12 课是自动化攻击的基础。第 13 课（多样本越狱）是互补的长度利用攻击。第 14 课（ASCII 艺术/视觉攻击）是编码攻击。第 15 课（间接提示词注入）是 2026 年的生产攻击面。第 16 课涵盖防御工具链的对应部分（Llama Guard、Garak、PyRIT）。

## 动手实践

`code/main.py` 构建了一个玩具 PAIR 循环。目标是一个模拟分类器，会拒绝"明显"有害的提示词（关键词过滤器）。攻击者是一个基于规则的精炼器，尝试改写、角色扮演框架和编码等策略。评判者对响应评分。你可以观察攻击者在约 5-15 次迭代内成功绕过关键词过滤器，但在语义过滤器上失败。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| PAIR | "自动化越狱" | 提示词自动迭代精炼；攻击者 LLM + 评判者 LLM 的循环 |
| GCG | "梯度越狱" | 白盒词元级梯度搜索，寻找对抗性后缀 |
| 攻击成功率（ASR） | "k 次查询内越狱的百分比" | 主要指标；必须与查询预算和评判者身份一起报告 |
| 评判 LLM | "评分者" | 评判响应是否满足有害目标的 LLM |
| JailbreakBench | "评估基准" | 带有分类标签的标准化有害行为集 |
| HarmBench | "更大的基准" | 510 种行为，包含功能性和语义性危害测试 |
| TAP | "攻击树" | 带有分支 + 剪枝的 PAIR；以更高计算换取更高 ASR |
