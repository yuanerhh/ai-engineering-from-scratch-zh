# 智能体的共识与拜占庭容错

> 经典分布式系统 BFT 遇上随机 LLM。2025-2026 年涌现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）通过置信度探针对每票进行加权；**DecentLLMs**（arXiv:2507.14928）采用无领导者架构，使用并行工作者提案和几何中位数聚合；**WBFT**（arXiv:2505.05103）将加权投票与层次结构聚类相结合，将节点划分为核心节点和边缘节点。来自"AI 智能体能达成一致吗？"（arXiv:2603.01213）的实证结果令人警醒——即使是标量共识在今天也很脆弱——一个欺骗性智能体就能破坏混合智能体的共识。BFT 是必要条件，但不是充分条件。本课构建一个最小 BFT 协议，注入三种特定于智能体的攻击（拜占庭谎言、顺从性一致性、相关错误单一文化），并测量每种共识变体的应对能力。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 16 · 07（心智社会与辩论）、Phase 16 · 13（共享内存）
**预计时间：** 约 75 分钟

## 问题背景

你有 N 个 LLM 智能体，每个都产生一个答案。它们不一致。多数投票选出了错误的答案，因为两个智能体是相关的（相同的基础模型、相同的训练数据、相同的失败模式）。第三个智能体以一种新颖的方式出错——所以多数是假多数。

现在加入一个欺骗性智能体：它故意撒谎。或者一个顺从性智能体：它同意最后说话的人。在经典 BFT 中，假设是拜占庭节点是 `f < n/3` 的比例，行为是任意的。2026 年的现实是：LLM 节点即使诚实时也是随机的，跨模型相关，并受彼此输出的影响。你不能把它们视为独立的伯努利投票者。

经典 BFT（PBFT，1999）没有错——只是不完整。它处理任意位翻转。它不处理"三个诚实的智能体共享同一个幻觉，因为它们共享训练数据"。本课从 PBFT 的基础出发，并在其上添加三种 2025-2026 年的改进。

## 核心概念

### 经典 BFT 给你什么

实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 个拜占庭节点。该协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、法定证书）。在 `n >= 3f + 1` 个诚实或恶意节点之间就单个值达成协议。

保证很强，但假设：

1. **独立故障。** 拜占庭节点不协调。
2. **诚实节点确实诚实。** 诚实输出的正确性不是问题；协议只是对齐分歧。
3. **问题有真实答案。** 对错误事实的共识仍然是共识。

LLM 智能体违反了这三点。运行相同基础模型的两个智能体共享故障。"诚实的"LLM 仍然会产生幻觉。对于模糊的问题，"真相"是智能体决定的——没有外部神谕。

### 三种特定于 LLM 的攻击

**拜占庭谎言。** 一个智能体输出故意错误的答案。如果 `f < n/3`，经典 BFT 能处理这个。

**顺从性一致性。** 一个智能体在投票前阅读其他人的答案，并与最后说话的人对齐。不是恶意的，但与最响亮的声音相关联。经典 BFT 不能阻止这种情况，因为该智能体通过了所有签名检查。

**相关错误单一文化。** 三个智能体共享一个基础模型。它们幻觉出同样错误的答案。多数是错的。经典 BFT 没有帮助，因为所有三个都是"诚实地"同意。

### 2025-2026 年的应对措施

**CP-WBFT**（arXiv:2511.10400）— 置信度探针加权 BFT。每个投票者在其答案上附加一个置信度探针（自我报告的概率，或单独校准模型的预测）。投票权重随置信度缩放。报告 BFT 改进了 +85.71%（在完全图上）。缓解措施：顺从性一致性（顺从的智能体在其自愿立场上倾向于有低置信度）。

**DecentLLMs**（arXiv:2507.14928）— 无领导者。工作者智能体并行提案，评估者智能体对提案评分，最终答案是评分位置的几何中位数。当 `f < n/2` 时健壮。缓解措施：拜占庭谎言和相关错误（几何中位数对异常值健壮，向密集簇拉近，而不是模型偏向的均值）。

**WBFT**（arXiv:2505.05103）— 带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史中学习的信任分数分配。将智能体聚类为核心和边缘；核心智能体必须首先达成共识，边缘智能体随后跟随。缓解措施：可扩展性（核心共识小而快）和部分针对单一文化（核心可以选择多样性）。

### 实证："AI 智能体能达成一致吗？"（arXiv:2603.01213）

该论文测量了多个前沿模型之间的标量一致性（LLM 智能体在单个数值上达成一致）。发现令人不安：

- 即使没有对手，LLM 智能体在许多基准上对标量问题的分歧率超过 30%。
- 一个采用欺骗性角色的智能体可以将混合智能体共识从诚实基线拉偏 40 多个百分点。
- 分歧率与模型多样性相关——异构集成比同质集成分歧更多（好：不相关错误）但收敛更慢（坏：达成一致的时间更长）。

结论：BFT 给你提供了对齐输出的机制，但不告诉你对齐的输出是否正确。结合验证（Phase 16 · 08 角色专业化）、多样性（Phase 16 · 15 辩论变体）和评估者智能体（Phase 16 · 24 基准测试）。

### 核心协议（精简版）

用于 LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探针 c_i in [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义簇分组（等价答案）
5. 聚合器计算每个簇 C 的权重：
     w(C) = sum_{i in C} c_i
6. 获胜者 = 权重最大的簇，如果 max > threshold * sum(c_i)
   否则：重试或上报
7. 少数簇带溯源记录以供事后审计
```

语义聚类步骤是特定于 LLM 的关键。"研究报告 4.2%"和"4.2% 改进"是同一个簇。朴素的字符串相等检查会错过这个。在生产中，使用廉价的嵌入模型或明确的规范化。

### 阈值调整

`threshold` 参数决定何时接受以及何时重试。太低：你接受弱多数。太高：你永远不接受任何东西。经验范围：对于 `n=5-7` 个智能体，0.5-0.67，对于较小的 `n` 更高。低于阈值时，上报给人类或不同的智能体集成。

### 共识无效的场景

- **模糊问题。** 如果问题没有基准真相，共识是一个观点。这样称呼它。
- **复合问题。** "写代码并解释它"——两个答案。分别对每个投票。
- **对抗性多轮。** 如果智能体可以观察先前轮次并模仿（Du 2023 辩论），它们开始不管真相如何都互相同意。限制轮次（通常 2-3 轮）。

## 动手实践

`code/main.py` 实现了：

- `AgentVoter` — 带（答案、置信度）的脚本化策略。
- `MajorityVote` — 经典多数投票。
- `CPWBFT` — 带语义聚类的置信度加权投票。
- `DecentLLMs` — 对评分提案进行几何中位数聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，置信度匹配。
3. `monoculture`：三个智能体以中等置信度共享一个错误答案（相关错误）。

运行：

```
python3 code/main.py
```

预期输出：（攻击，聚合器）→ 最终答案的表格，正确答案高亮显示。多数投票在单一文化情况下失败。CPWBFT 的置信度加权缓解了顺从性。DecentLLMs 的几何中位数在单一文化少于人口一半时向诚实簇拉近。

## 产出技能

`outputs/skill-consensus-designer.md` 为多智能体集成设计共识协议：聚类方法、加权、阈值，以及低于阈值轮次的上报策略。

## 输出

在发布任何共识机制之前：

- **至少用上面三种模式进行攻击测试。** 你的协议应该可预测地失败，而不是悄悄地失败。
- **记录每个少数簇**并附溯源。少数簇是你发现相关错误的预警系统。
- **强制执行有界轮次。** 不允许"继续辩论直到达成一致"——那会奖励顺从性。
- **将一致性与正确性分开。** 共识输出去往验证者；验证者独立于集成。
- **监控一致率。** 急剧上升意味着一致性偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认多数投票在单一文化攻击下失败，但当单一文化置信度低于 0.7 时，CPWBFT 部分缓解它。
2. 添加第四种攻击模式：**沉默弃权**——一个智能体拒绝回答（"我不知道"）。每个聚合器应该如何对待弃权？实现你的选择。
3. 将语义聚类从字符串规范化换成嵌入相似度（使用任何开源嵌入模型）。顺从性攻击会发生什么？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探针校准步骤（一个单独的校准模型检查每个智能体的自我报告置信度）。测量单一文化场景的准确性提升。
5. 阅读"AI 智能体能达成一致吗？"（arXiv:2603.01213）。复现一个简化的标量一致性实验：三个智能体，一个标量问题，欺骗性角色提示词。CPWBFT 或 DecentLLMs 能捕获它吗？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，用于容忍 `f < n/3` 任意故障的共识。 |
| 拜占庭（Byzantine） | "任何坏行为" | 可以撒谎、丢弃消息、悄悄失败的节点——除了安全崩溃外的任何事。 |
| 置信度探针（Confidence probe） | "你有多确定？" | 附加到投票上的自我报告或校准器预测的概率。 |
| 语义聚类（Semantic clustering） | "同一答案，不同措辞" | 在计票之前对等价答案进行分组。 |
| 几何中位数（Geometric median） | "健壮中心" | 最小化到样本点距离之和的点。对异常值健壮，不像均值。 |
| 单一文化（Monoculture） | "相同模型，相同失败" | 当智能体共享训练数据或基础模型时的相关错误。 |
| 顺从性一致性（Sycophantic conformity） | "同意大声的声音" | 智能体的投票偏向最先/最响亮发言的人。 |
| 核心/边缘（Core/Edge） | "层次化 BFT" | WBFT 分割：先进行小型核心共识，边缘节点随后跟随。限制延迟。 |
