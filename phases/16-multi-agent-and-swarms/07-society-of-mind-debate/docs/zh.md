# 心智社会与多智能体辩论

> Minsky 1986 年的前提——智能是专家社会——每十年被重新发现一次。2023 年 Du 等人将其转化为一个具体算法：多个 LLM 实例提出答案，阅读彼此的答案，批评，并更新。经过 N 轮，它们在六个推理和事实性任务上收敛到超越零样本 CoT 和反思的共识。两个发现很重要：**多个智能体**和**多轮**都独立贡献。社会胜过单智能体独白；多轮交流胜过一次性投票。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 16 · 04（原语模型）
**预计时间：** 约 60 分钟

## 问题背景

自我一致性——多次采样一个模型并取多数答案——是你可以附加的最便宜的推理改进。它有效，但很快饱和。你可以将样本数量翻倍却看不到另一个有意义的跳跃。

辩论打破了饱和。不是一个模型的 N 个独立样本，而是 N 个智能体互相阅读推理并修正。样本之间的相关性降低（它们不再是独立同分布的），收敛点通常在独立同分布投票自信地出错的地方是正确的。

## 核心概念

### Du 等人 2023 年算法

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体中的每一个对问题产生初始答案。
2. 对于轮次 r = 2..R：每个智能体被展示其他智能体的第 r-1 轮答案，并被问"考虑这些，给出你的更新答案"。
3. R 轮后，对最终答案进行多数投票。

论文在 MMLU、GSM8K、传记、MATH 和事实性基准上测试。辩论始终胜过 CoT 和自我反思。

### 两个独立的旋钮

来自同一论文的消融实验：

- **单独智能体数量**（1 轮，N 的多数投票）在大多数任务上胜过单智能体，但会饱和。
- **单独轮数**（1 个智能体看到自己的先前推理）几乎没有帮助——这是反思的已知弱点。
- **两者同时**产生大跳跃。多个智能体之间的多轮交流驱动了增益。

### 为什么有效

两个机制：

1. **暴露于分歧。** 当一个智能体看到另一个智能体带有不同结论的推理链时，它必须要么辩护要么更新。无论如何，第 r+1 轮的上下文比第 r 轮更丰富。
2. **相关错误减少。** 在自我一致性中，所有样本来自同一个模型，因此错误相关——你平均到一个自信的错误答案。不同的模型或不同的随机种子去相关。不同的*辩论观点*进一步去相关。

### 异质辩论

A-HMAD 和相关后续工作对不同的智能体使用*不同的基础模型*。Llama + Claude + GPT 辩论减少了单一文化崩溃（第 26 课），因为一个模型家族的相关错误不被其他模型共享。

缺点：参与辩论的弱模型可以将共识拖向其错误答案（参见"我们应该去 MAD 吗？"，arXiv:2311.17371）。

### NLSOM — 129 智能体扩展

Zhuge 等人（"基于自然语言的心智社会中的风暴"，arXiv:2305.17066）将这个思想扩展到 129 成员的社会。结果：专业化和自组织随规模出现，系统在视觉问答等任务上胜过单智能体。

### 失败模式

- **顺从级联（Sycophancy cascade）。** 所有智能体都服从听起来最自信的那个智能体。辩论崩溃为最大声的声音。提示对抗性角色（"一个智能体必须争论反对立场"）有帮助。
- **话题漂移。** 许多轮次的辩论从原始问题漂移。缓解措施：每轮重新注入问题。
- **计算爆炸。** N 个智能体 × R 轮 = N·R 次 LLM 调用，每次都有增长的上下文。5 智能体、5 轮辩论是 25 次调用，上下文增长。每个问题的成本可以超过单次 CoT 调用的 10 倍。

## 动手实践

`code/main.py` 对一个数学问题运行 3 智能体 × 3 轮辩论，每个智能体以不同的（可能错误的）答案开始。智能体是脚本化的——每个通过以脚本化置信度加权邻居答案来"更新"。收敛在逐轮日志中可见。

演示显示两个关键效果：

- 一轮交流将智能体移近正确答案。
- 第 2 轮之后的额外轮次显示收益递减（与 Du 等人的平台期匹配）。

运行：

```
python3 code/main.py
```

## 产出技能

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮数、异质性（相同模型与混合）、角色分配（对称与一方对抗）。它还在运行前估计 token 成本。

## 输出

如果你发布辩论：

- **将轮数限制在 3。** Du 等人显示 3 轮捕获了大部分增益。更多是成本，不是质量。
- **将智能体限制在 5。** 超过 5 个，上下文膨胀和成本主导。
- **默认异质。** 池中至少两种不同的基础模型。
- **对抗槽位。** 一个智能体被提示无论如何都要反对。打破顺从性。
- **记录每一轮。** 隐藏中间轮次的辩论系统无法调试或审计。

## 练习

1. 运行 `code/main.py`，然后将轮数设置为 5 并观察收益递减。在哪一轮额外的收敛停止？
2. 添加第四个具有对抗性角色的智能体：总是与当前多数不同意。这会破坏还是改善收敛？
3. 绘制（打印）每轮的同意分数（在多数答案上的智能体比例）。它什么时候达到 1.0，这是否等同于"正确"？
4. 阅读 Du 等人第 4 节消融实验。使用此代码复制"仅智能体"vs"仅轮次"vs"两者"的结果。
5. 阅读"我们应该去 MAD 吗？"（arXiv:2311.17371），列出超越轮询的两种辩论变体——例如，法官主导、辩论链、对抗性。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 心智社会（Society of Mind） | "Minsky 的思想" | 智能作为交互专家；1986 年框架现在通过 LLM 辩论操作化。 |
| 多智能体辩论（Multi-agent debate） | "智能体争论" | N 个智能体提议，互相批评，R 轮后修改，多数投票。 |
| 共识（Consensus） | "他们同意" | 不是认知真理——只是在多数答案上的比例。可以是自信地错误的。 |
| 轮次（Rounds） | "交换步骤" | 一轮 = 每个智能体阅读其他人并更新一次。 |
| 异质辩论（Heterogeneous debate） | "混合模型家族" | 使用不同的基础模型来去相关错误。 |
| 顺从级联（Sycophancy cascade） | "每个人都同意大声的那个" | 辩论失败，智能体服从最自信的智能体而不管正确性。 |
| NLSOM | "129 智能体社会" | 基于自然语言的心智社会；Zhuge 等人的扩展版本。 |
| 相关错误（Correlated error） | "相同模型，相同错误" | 为什么自我一致性饱和；跨不同观点的辩论去相关。 |
