# 投票、自一致性与辩论拓扑

> 最便宜的聚合方式：采样 N 个独立智能体，多数投票。Wang 等人 2022 年的自一致性对单个模型采样 N 次实现了这一点。多智能体通过**异构**智能体将其扩展，以逃脱单一文化——不同的模型、不同的提示词、不同的温度、不同的上下文。除了多数投票之外，辩论拓扑也很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星形/链式/树形/图形协调，发现**图形拓扑最适合研究**，且在约 4 个智能体后存在"协调税"。AgentVerse（ICLR 2024）记录了两种涌现模式——志愿行为和顺从行为——顺从既是特性（找到共识）也是风险（群体思维，第 24 课）。本课绘制拓扑空间，构建每种变体，并测量协调税。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 16 · 07（心智社会与辩论）、Phase 16 · 14（共识与 BFT）
**预计时间：** 约 75 分钟

## 问题背景

辩论可以提高准确性（Du 等人，arXiv:2305.14325）。也可以降低准确性。辩论是否有帮助取决于四个结构性选择：

1. 谁与谁交流（拓扑）。
2. 多少轮（Du 2023：轮次和智能体数量独立地都很重要）。
3. 智能体是否是异构的（不同的基础模型打破单一文化）。
4. 是否存在对抗性声音（钢人论证 vs 稻草人论证）。

那些简单地"运行 5 个智能体并投票"的团队通常比单个智能体表现更差。失败不是随机的。它们跟踪拓扑和异构性。本课是拓扑地图。

## 核心概念

### 自一致性：单模型基线

Wang 等人 2022 年（"自一致性改善思维链推理"）以温度 > 0 对同一模型采样 N 次，并对推理路径答案进行多数投票。GSM8K 上的结果：N=40 次采样相比单次贪婪解码有显著提升。自一致性是多智能体投票的单智能体前驱。

限制：自一致性使用一个基础模型。错误是由构造相关的。如果模型有系统性偏差，所有 N 个样本都共享它。

### 多智能体投票：异构扩展

将 N 个样本替换为 N 个*不同的*智能体。不同的基础模型（Claude、GPT、Llama）、不同的提示词、不同的工具访问。好处：不相关的错误。代价：不同的智能体成本不同；协调它们增加了开销。

2026 年对异构辩论的规范名称是 **A-HMAD**——对抗性异构多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。不是普遍采用的，但论文用这个术语表示"不同模型辩论，减少来自单一文化崩溃的相关错误"。

### 四种拓扑

```
星形                链式               树形                图形

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           （全连接）
```

星形：一个中心，其他所有人只与中心交流。等同于没有反向通道的监督者-工作者。
链式：线性，每个智能体看到前一个智能体的输出。类似流水线。
树形：层次化，用于层次智能体系统（第 06 课）。
图形：任意到任意。包括全连接团和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包括研究、编码和规划的任务套件上对星形、链式、树形、图形拓扑进行了基准测试。关键测量结果：

- **图形**拓扑在研究任务上获胜。信息流向任意方向；智能体可以互相批评。
- **星形**在快速回答事实性任务上获胜。中心过滤和整合。
- **链式**在逐步流水线（分阶段精炼）上获胜。
- **协调税**在图形拓扑中在约 4 个智能体之后出现。挂钟时间和 token 成本比质量增长更快。

4 智能体上限是经验性的，不是基本性的。它反映了 2026 年 LLM 的上下文容量：每个智能体的上下文充满了对等体的输出，一旦每个人都能看到所有人，添加第 N+1 个智能体的边际价值就会下降。

### 多智能体辩论策略（"我们应该走向 MAD 吗？"）

arXiv:2311.17371 是 2023 年对 MAD 策略的调查。其他人复制的关键发现：与自一致性*结构相似*的 MAD 变体（独立采样 + 聚合）在相同预算下通常不如自一致性。MAD 在智能体确实异构且辩论有对抗性结构时最有帮助（一个智能体反对论证）。

### AgentVerse 的涌现模式

AgentVerse（ICLR 2024）记录了即使没有明确设计，多智能体辩论中涌现的两种行为：

- **志愿。** 一个智能体自发地提供帮助（"我可以做下一步"）。有用：它将工作分配给最有能力处理子任务的智能体。
- **顺从。** 一个智能体调整其立场以匹配批评者，即使批评者是错误的。这是辩论中顺从性（第 14 课）的等价物。

顺从性是为什么辩论到达成一致会奖励欺凌者。有独立裁判的有界轮次可以缓解这个问题。

### 异构性：实际移动准确性的旋钮

实践文献中的 2024-2026 模式：将 N 个智能体中的一个换成不同的基础模型，比将 N 增加 1 带来更大的准确性提升。直觉是单一文化——每个新的独立错误源的价值超过一个额外的相关样本。

在极限情况下，异构性胜过数量。三个不同模型在大多数有干净基准真相的任务上胜过一个模型的五个副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中引用）将"陪审团"形式化——一小组专业智能体通过在每个阶段投票来精炼答案。与纯多数投票不同，陪审团有角色：一个智能体交叉审问，一个提供上下文，一个评分可信度。陪审团方法是普通投票（廉价，单一文化倾向）和完整 MAD（昂贵，顺从倾向）之间的中间点。

### 辩论投票占优的场景

- 问题有基准真相（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同的来源或工具（异构性可用）。
- 轮次有界（通常 2-3 轮），并有独立的裁判或验证者。
- 预算允许 3-5 个智能体。在图形拓扑中超过 5-7 个，协调税占主导。

### 辩论投票有害的场景

- 问题是意见型的。智能体收敛到看起来最自信的答案，而不是最正确的答案。
- 所有智能体共享一个基础模型。单一文化使共识毫无意义。
- 轮次无界。每次顺从性都会获胜。
- 任务很简单。N=5 时带自一致性的单个智能体更便宜且同样准确。

## 动手实践

`code/main.py` 实现了：

- `run_star(agents, hub, question)` — 中心轮询每个工作者，聚合。
- `run_chain(agents, question)` — 顺序精炼。
- `run_tree(root, children, question)` — 带深度 2 聚合的层次化。
- `run_graph(agents, question, rounds)` — 全对全辩论，有界轮次。
- 脚本化异构性刻度盘：每个智能体有一个 `error_bias`，表明其系统性错误。
- 一个测量工具，在 N=3、5、7 时运行每种拓扑，并报告（准确性、总 token、模拟挂钟时间）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N → （准确性、token、延迟）的表格。图形在 N=3-5 时在研究风格任务上获胜；星形在快速事实任务上获胜；N=7 时的图形显示了协调税（延迟比准确性增长更快）。

## 产出技能

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星形/链式/树形/图形）、N（智能体数量）、异构性配置（要使用的基础模型）和轮次上限。

## 输出

对于任何集成：

- 从**N=5 时使用一个强基础模型的自一致性**开始。这是廉价的基线。
- 如果准确性重要，升级到 **N=3 时的异构投票**。测量差异。
- 只有在任务有结构（研究、多步骤）且有界轮次可行时，才升级到**辩论拓扑**。
- 始终记录少数簇。当少数簇持续正确时，你有一个多样性信号。
- 将挂钟时间和 token 与准确性一起进行基准测试。"以 10 倍成本提高准确性"是一个商业决策。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 自一致性（Self-consistency） | "采样 N 次，投票" | Wang 2022。单个模型，N 次温度>0 的采样，对推理路径的多数投票。 |
| 异构性（Heterogeneity） | "不同模型" | 不同基础模型或提示词族的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体在多轮次中交换批评的通用术语。参见 Du 2023。 |
| A-HMAD | "对抗性异构 MAD" | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| 拓扑（Topology） | "谁与谁交流" | 星形、链式、树形、图形。决定信息流。 |
| 协调税（Coordination tax） | "收益递减" | 在图形拓扑中约 4 个智能体以上，成本比质量增长更快。 |
| 志愿行为（Volunteer behavior） | "主动帮助" | AgentVerse 涌现模式：一个智能体自发提出接下一步。 |
| 顺从行为（Conformity behavior） | "压力下的同意" | AgentVerse 涌现模式：一个智能体与批评者对齐。 |
| 陪审团（Jury） | "小型专业小组" | Sibyl 风格的集成，带角色（审问者、上下文提供者、评分者）。 |
