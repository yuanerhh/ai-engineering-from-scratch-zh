# 智能体经济、代币激励、声誉

> 长视野自主智能体（METR 的 1 小时到 8 小时工作曲线）需要经济自主权。新兴的**五层堆栈**是：**DePIN**（物理计算）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产智能体激励网络包括 **Bittensor**（TAO 子网奖励特定任务模型）、**Fetch.ai/ASI 联盟**（ASI-1 Mini LLM + FET 代币）和 **Gonka**（基于 Transformer 的 PoW，将计算重新分配给有生产力的 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 价值信用归因**来公平奖励贡献智能体；Google Research "大型语言模型的机制设计"在单调聚合下提出了带第二价格支付的**代币拍卖**。本课构建一个最小的智能体市场，将 Shapley 价值信用归因应用于多智能体管道，并运行第二价格代币拍卖，使博弈论机制具体落地。

## 问题背景

当智能体共同产生价值但需要单独奖励时，多智能体系统变得复杂。经典机制——平均分配、最后贡献者通吃——是不公平或可被游戏的。通过 Shapley 值进行基于联盟的奖励在构造上是公平的，但计算成本高昂。2025-2026 年的文献推动了有用的近似：Shapley 采样、单调聚合拍卖和从确认贡献中积累的链上声誉。

超越信用归因，该领域已转向实际经济智能体：Bittensor TAO 奖励挖矿计算以微调子网特定模型，Fetch.ai/ASI 用 FET 代币奖励 ASI-1 Mini LLM 使用，Gonka 将 Transformer 工作量证明重新分配给有生产力的 AI 任务。今天已经存在自主交易的智能体；问题是如何对齐激励。

本课将智能体经济视为一个特定的问题家族——信用归因、机制设计和声誉——并用最少的数学构建每个，使思想得以坚固。

## 核心概念

### 五层智能体经济堆栈

1. **DePIN（物理计算）。** 租用 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。不特定于智能体；智能体使用它。
2. **身份。** W3C 去中心化标识符（DID）给每个智能体提供独立于任何平台的持久 ID。声誉积累到 DID。智能体网络协议（ANP）使用 DID 作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）让智能体从自己的余额支付 gas，而无需持有 ETH。智能体可以为服务、彼此或计算付款。
5. **治理。** 智能体 DAO：人类*和*智能体对协议变更投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都使用所有五层。Bittensor 使用第 1、2 层，部分第 3、4 层，不使用第 5 层。OpenAI 智能体除第 3 层外什么都不使用。这个堆栈是参考地图，不是要求。

### Bittensor、Fetch.ai、Gonka——正在运行的内容

**Bittensor（TAO）。** 子网是专业任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对其排名；质押加权评分分配 TAO 奖励。每个子网有自己的评估。经济教训：为特定任务输出质量付款，而不是使用的计算。

**Fetch.ai/ASI 联盟。** ASI-1 Mini LLM 在 Fetch.ai 的网络上运行；用户为推理支付 FET 代币。智能体作为对等体的叙述在这里更强：Fetch 上的智能体可以为任务调用另一个智能体并以 FET 付款。

**Gonka。** Transformer 工作量证明："工作"是 Transformer 的前向传递。矿工通过运行具有已知正确输出（来自训练数据）的推理任务赚钱。基于资源生产力的 PoW，而不是基于哈希的 PoW。

截至 2026 年 4 月，三者都是生产级的。收益分配不同。Bittensor 相对于子网验证者奖励质量；Fetch 奖励由付费用户衡量的效用；Gonka 奖励可验证的推理工作。

### Shapley 价值信用归因

三个智能体协作完成任务。输出得分 0.8。谁贡献了什么？

Shapley 值：满足四个公理（效率、对称性、线性、空性）的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * 所有排序 O 的总和 (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是排序 `O` 中在 `i` 之前的智能体集合。在实践中：枚举所有排列，记录每个智能体在每个排列中的边际贡献，取平均值。

对于 N=3 个智能体，有 6 个排列。对于 N=10，有 360 万个——所以在实践中你对排序进行采样而不是枚举。

### 用于聚合的第二价格拍卖

Google Research（"大型语言模型的机制设计"）为聚合 LLM 输出提出第二价格代币拍卖。设置：N 个智能体各自提出一个补全；每个都有被选中的私人价值。拍卖师选择最高价值的提案并支付*第二高*价值。在单调聚合下（价值取决于选择哪个提案，而不是有多少投标），这是真实的——智能体以真实价值投标。

这对 LLM 系统为什么重要：你可以将补全任务外包给具有不同定价的多个智能体；拍卖选择最好的 + 公平支付，智能体没有动机误报。

### 声誉资本

与 DID 绑定的声誉分数从确认的贡献中积累。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 对路由决策的读取成本低廉（"将困难任务发送给高声誉智能体"）。
- 伪造成本高昂（随时间积累，与 DID 绑定）。
- 可以被削减：未能通过验证的贡献会减去。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 价值信用归因和简单的拍卖机制。关键主张：去中心化信用归因步骤使系统可审计并且免疫单点操纵。

### 经济学失效的场景

- **价格预言机操纵。** 如果信用函数可以被游戏，智能体会游戏它。每种机制都需要对抗性测试。
- **Sybil 攻击。** 一个操作者创建 N 个假智能体来夸大自己的贡献。DID 减缓但不能阻止这个；伪造声誉的成本是缓解措施。
- **验证成本。** 信用归因只有在验证者公正时才公平。如果验证成本低廉（小型 LLM），可以被游戏；如果昂贵（人类小组），系统无法扩展。
- **监管悬而未决。** 智能体经济与金融监管交叉。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法管辖区都在法律灰色地带运营。

### 智能体经济有意义的场景

- **具有异构运营者的开放网络。** 没有单一团队控制所有智能体。
- **可验证的输出。** 没有验证，信用归因只是猜测。
- **长视野工作流。** 一次性任务不受益于声誉积累。
- **代币化支付在你的司法管辖区合法可行。**

在封闭的企业系统中，经济学让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 输出

2026 年运行智能体经济：

- **从声誉开始，不是代币。** 声誉实现成本低廉且单独有价值；代币增加了法律和经济复杂性。
- **在奖励之前验证。** 永远不要在没有独立验证步骤的情况下分配信用。自我报告的质量会积累 Sybil 游戏。
- **Shapley 采样，而不是 Shapley 精确计算。** 采样 100-1000 个排序；精确枚举无法扩展。
- **限制衰减因子和声誉下限。** 无限衰减会清除合法贡献者；太慢的衰减会奖励陈旧的高声誉智能体。
- **对抗性审计机制。** 在开放网络之前运行红队场景。每种机制都有博弈论；你想在攻击者之前找到漏洞。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| DePIN | "去中心化物理基础设施" | 代币激励的计算/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | W3C 规范，用于可携带 ID。智能体声誉与 DID 绑定，而不是平台。 |
| ERC-4337 | "账户抽象" | 可以赞助 gas 的合约账户，使智能体支付成为可能。 |
| Shapley 值（Shapley value） | "公平信用归因" | 满足效率、对称性、线性、空性的唯一分配。 |
| 第二价格拍卖（Second-price auction） | "Vickrey 拍卖" | 真实机制：获胜者支付第二高出价。与单调聚合兼容。 |
| 声誉资本（Reputation capital） | "累积质量分数" | 来自确认贡献的 DID 绑定分数；随时间衰减。 |
| 智能体 DAO（Agentic DAO） | "智能体 + 人类治理" | 智能体投票者为一等公民、投票权与声誉挂钩的 DAO。 |
| TAO / FET / GPU 积分 | "代币面值" | Bittensor TAO、Fetch.ai FET、各种 DePIN 代币。 |
