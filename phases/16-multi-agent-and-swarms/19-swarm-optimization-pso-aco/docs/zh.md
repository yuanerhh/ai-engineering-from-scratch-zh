# LLM 的群体优化（PSO、ACO）

> 生物启发优化正在 LLM 领域卷土重来。**LMPSO**（arXiv:2504.09247）使用 PSO，其中每个粒子的速度是一个提示词，LLM 生成下一个候选者；在结构化序列输出（数学表达式、程序）上效果很好。**Model Swarms**（arXiv:2410.11163）将每个 LLM 专家视为模型权重流形上的 PSO 粒子，在 9 个数据集上相比 12 个基线**平均提升 13.3%**，仅用 200 个实例。**SwarmPrompt**（ICAART 2025）将 PSO + 灰狼算法混合用于提示词优化。**AMRO-S**（arXiv:2603.12933）是 ACO 启发的多智能体 LLM 路由信息素专家——**4.7 倍加速**，可解释的路由证据，将推理与学习解耦的质量门控异步更新。本课在提示词参数空间上实现 PSO，在智能体路由上实现 ACO，测量这些经典算法为什么适合 LLM 时代，以及什么时候不适合。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 16 · 09（并行群体网络）、Phase 16 · 14（共识与 BFT）
**预计时间：** 约 75 分钟

## 问题背景

你有一个在任务评估上得分 62% 的提示词。你想改进它。朴素的做法是无梯度的手动调整，这扩展性很差。强化学习需要奖励信号和足够的展开次数来训练。通过提示词反向传播实际上是不可能的——提示词是一个离散字符串，不是可微参数。

经典的生物启发优化——PSO 用于连续搜索空间，ACO 用于路径选择——正是为这种情况设计的：无梯度、基于种群、每次评估成本低廉。将它们与 LLM 配对进行无梯度搜索步骤，你会得到一个出人意料地实用的优化器。

同样的模式适用于多智能体系统中的智能体*路由*。ACO 风格的信息素轨迹记录哪个智能体在哪种任务类型上表现最好，让路由器利用该轨迹，并衰减信息素以便可以重新发现路由。

## 核心概念

### PSO 回顾（Kennedy & Eberhart 1995）

粒子群优化：连续搜索空间中的粒子种群。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
if improved: update p_best_i
if global best: update g_best
```

其中 `p_best` 是粒子自身的最佳，`g_best` 是群体的最佳，`w, c1, c2` 是惯性 + 认知 + 社会权重，`r1, r2` 是随机因子。

### LLM 输出上的 PSO——LMPSO

arXiv:2504.09247 将 PSO 适应于 LLM 生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个*提示词*，描述如何将当前输出向个人/全局最佳修改。LLM 从速度提示词生成新输出。速度的"惯性"是像"做小增量变化"这样的提示词。

这在以下情况效果很好：
- 输出是结构化的（可解析、可评估）。
- 适应度是自动的（测试运行、算术评估）。
- 种群很小（约 10-30 个粒子），使总 LLM 调用保持可管理。

当适应度需要人工审查时效果不好——每次迭代的成本变得过高。

### Model Swarms

arXiv:2410.11163 将 PSO 从输出层移到*模型*层。每个"粒子"是一个专家 LLM（参数）。群体通过无梯度更新将参数向集体最佳移动。报告：在 9 个数据集上相比 12 个基线平均提升 13.3%，每次迭代仅用 200 个实例。

关键洞见是 LLM 专家模型已经在共享参数流形附近（适配器权重、LoRA 增量）。在这个低维子空间上的 PSO 廉价而有效。

### ACO 回顾（Dorigo 1992）

蚁群优化：蚂蚁遍历图形；每条路径有一个信息素轨迹。蚂蚁的移动概率按信息素强度加权。完成任务的蚂蚁按解决方案质量比例沉积信息素。信息素随时间衰减。

### AMRO-S——用于智能体路由的 ACO

arXiv:2603.12933 使用 ACO 进行多智能体路由。每种任务类型是一个"目的地"；每个智能体是一条可能的路由。信息素加强产生良好输出的路由。关键贡献：

- **可解释的路由证据。** 信息素强度是人类可读的信号。
- **质量门控异步更新。** 只有在质量检查通过后才更新信息素，将推理与学习解耦。
- **4.7 倍加速**，在多智能体路由基准上。

质量门很重要：没有它，快速但错误的智能体会积累信息素，系统会锁定在糟糕的路由上。

### 何时为 LLM 使用 PSO / ACO

**使用 PSO 当：**
- 搜索空间是连续的或映射到连续参数（提示词嵌入、LoRA 权重、数值生成参数）。
- 适应度廉价且自动。
- 种群可以很小（10-30）。

**使用 ACO 当：**
- 你有路由或路径选择问题。
- 决策随时间强化（相同的任务类型重复出现）。
- 你需要可解释的路由决策证据。

**不要使用任何一个当：**
- 适应度需要人工审查（每次迭代成本太高）。
- 搜索空间是离散的且是 PSO 无法覆盖的组合方式（改用遗传算法）。
- 实时决策需要严格的延迟（PSO/ACO 相对于单次启发式算法收敛很慢）。

### 为什么生物启发仍然获胜

基于梯度的方法需要可微信号。LLM 输出和路由决策不是容易可微的。伪梯度方法（强化学习的路由器、DPO 风格的提示词调整器）有效，但需要昂贵的训练。

PSO 和 ACO 只需要一个*评估器*函数。如果你可以对候选输出或路由决策评分，你就可以在空间上进行优化。这使适用性的门槛低得多。

### 实际限制

- **种群预算。** N 个粒子 × T 次迭代 × 每次评估成本。对于每次调用约 0.02 美元的 LLM 评估，运行 50 次迭代的 20 粒子 PSO 花费约 20 美元。相应地规划。
- **探索 vs 利用。** 信息素衰减率和 PSO 惯性之间权衡；衰减太快 → 忘记解决方案；太慢 → 困在早期局部最优。
- **灾难性漂移。** 如果适应度景观改变（新的数据分布），两种算法都可能收敛后再发散。监控最佳适应度稳定性。

## 动手实践

`code/main.py` 实现了：

- `LMPSO` — 数值提示词参数上的 PSO（温度、top_k 权重）。每个粒子的"LLM 生成"被模拟为脚本化适应度函数。运行算法 30 次迭代并显示 g_best 收敛。
- `AMRO_S` — ACO 风格路由。3 个智能体，4 种任务类型，信息素矩阵，100 个路由任务。打印随时间变化的（任务类型 → 智能体选择）分布以显示轨迹形成。
- 比较：在相同任务流上的随机路由 vs ACO 路由。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适应度在 30 次迭代中从随机提升到接近最优。
- AMRO-S：信息素表在每种任务类型的正确智能体上稳定；ACO 路由在质量上比随机路由好约 30-40%，也减少了延迟（重试更少）。

`outputs/skill-swarm-optimizer.md` 帮助在 PSO、ACO、遗传算法和基于梯度的优化器之间为 LLM/智能体优化问题做出选择。

## 输出

- **从小开始。** 10-20 个粒子，20-50 次迭代。只有当收敛曲线显示明显增益时才扩大规模。
- **记录每次迭代的信息素或 g_best。** 在没有轨迹的情况下调试群体优化器很痛苦。
- **质量门控更新。** 特别是对于 ACO 路由：快速但错误的智能体不能积累信息素。
- **在分布偏移时重置衰减。** 当你的评估分布改变时，老化的信息素是陈旧的；重置或暂时加倍衰减率。
- **设置每次迭代成本上限。** 发布每次迭代成本指标。每次迭代花费 500 美元且增益 0.5% 的 PSO 是不可发布的。

## 练习

1. 运行 `code/main.py`。观察 LMPSO 收敛。变化种群大小 5、10、20、50。在哪个大小时收敛时间饱和？
2. 实现"灾难性漂移"实验：在第 30 次迭代后改变适应度函数。PSO 多快能适应？重置 `p_best` 有帮助吗？
3. 向 AMRO-S 添加质量门：只在评估分数 > 0.7 的运行上沉积信息素。这如何改变相对于无门版本的收敛？
4. 阅读 LMPSO（arXiv:2504.09247）。将论文的"速度作为提示词"映射回你的数值速度。在仿真中什么丢失了，什么保留了？
5. 阅读 AMRO-S（arXiv:2603.12933）。实现带异步信息素更新的解耦"推理快速路径"。这如何在持续负载下改变系统延迟？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| PSO | "粒子群优化" | Kennedy-Eberhart 1995。基于种群的无梯度优化器。 |
| ACO | "蚁群优化" | Dorigo 1992。通过信息素轨迹的路径/路由优化。 |
| LMPSO | "带 LLM 生成的 PSO" | arXiv:2504.09247。速度是提示词；LLM 产生候选者。 |
| Model Swarms | "专家权重上的 PSO" | arXiv:2410.11163。模型参数子空间上的无梯度更新。 |
| AMRO-S | "用于智能体路由的 ACO" | arXiv:2603.12933。任务类型 × 智能体上的信息素矩阵。 |
| p_best / g_best | "个人/全局最佳" | 每个粒子和整个群体迄今为止找到的最佳解决方案。 |
| 信息素（Pheromone） | "路由记忆" | 边上的强度；随时间衰减；在质量上沉积。 |
| 质量门控更新（Quality-gated update） | "只从好的运行中学习" | 以质量检查为条件的信息素沉积。 |
| 灾难性漂移（Catastrophic drift） | "分布偏移" | 适应度景观改变；旧的 p_best 和信息素变得陈旧。 |
