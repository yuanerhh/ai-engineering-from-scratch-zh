# 谈判与讨价还价

> 智能体对资源、价格、任务分配和条款进行谈判。2026 年的基准测试集很清楚：NegotiationArena（arXiv:2402.05863）显示 LLM 可以通过角色操纵（"绝望"）将收益提升约 20%；"测量语言模型的讨价还价能力"（arXiv:2402.15813）显示买家比卖家更难，规模没有帮助——他们的 **OG-Narrator**（确定性报价生成器 + LLM 叙述者）将成交率从 26.67% 推到 88.88%；大规模自主谈判竞赛（arXiv:2503.06416）进行了约 18 万次谈判，发现**思维链隐藏**智能体通过向对方隐藏推理而获胜；Bhattacharya 等人 2025 年基于哈佛谈判项目指标对 Llama-3 最有效、Claude-3 最具攻击性、GPT-4 最公平进行了排名。本课实现合同网协议（FIPA 祖先，第 02 课），连接 LLM 风格的买家/卖家，运行 OG-Narrator 风格的分解，并测量成交率如何随每个结构性选择变化。

**类型：** 学习 + 构建
**编程语言：** Python（标准库）
**前置知识：** Phase 16 · 02（FIPA-ACL 遗产）、Phase 16 · 09（并行群体网络）
**预计时间：** 约 75 分钟

## 问题背景

两个智能体需要就价格达成一致。在仅使用纯语言提示词的情况下，2024-2026 年的 LLM 以出人意料的低成交率（在 arXiv:2402.15813 的严格参数化讨价还价中约 27%）完成交易。规模无法解决这个问题：GPT-4 在结构上并不比 GPT-3.5 更善于讨价还价；它只是在讨价还价的*语言*上更好。

根本问题是 LLM 将两项工作混为一谈——决定报价和叙述报价。OG-Narrator 将这两者分开：确定性报价生成器计算数值动作；LLM 只负责叙述。成交率跳升至约 89%。

这映射了一个经典的多智能体发现：将机制与通信层解耦能获胜。合同网协议（FIPA，1996；Smith，1980）是参考任务市场机制。将 LLM 插入叙述槽位，你就获得了一个现代 LLM 驱动的任务市场。

## 核心概念

### 合同网，一段话概述

Smith 1980 年的合同网协议：一个**管理者**广播**提案请求（cfp）**；**投标者**用包含其报价的**提案**消息回应；管理者选出获胜者，向获胜者发送**接受提案**，向失败者发送**拒绝提案**。获胜者执行工作。可选消息：**拒绝**（投标者拒绝提案）。FIPA 将其编码为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 获胜

"测量语言模型的讨价还价能力"（arXiv:2402.15813）观察到：

- LLM 经常违反讨价还价规则（以荒谬的价格报价，忽略对方的 ZOPA）。
- 它们的锚定很差（接受糟糕的首次报价；以象征性而非策略性的金额还价）。
- 规模单独无法解决这些问题。更大的模型产生更合理的语言，但战略错误相似。

OG-Narrator 分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  状态   → │   报价生成器     │ 价格 → │    LLM 叙述者    │ → 消息
           │  （确定性）      │        │  （写人类风格    │
           │                  │        │   的附带说明）   │
           └──────────────────┘        └──────────────────┘
```

报价生成器是一种经典谈判策略：Rubinstein 讨价还价模型、Zeuthen 策略或简单的价格以牙还牙。LLM 进行叙述。消息包含确定性价格和自然语言框架。

成交率提升是因为：
- 价格保持在讨价还价区域内。
- 锚定是策略性的，不是情绪性的。
- LLM 做它擅长的事：写作。

### NegotiationArena 发现

arXiv:2402.05863 提供了规范基准。标志性发现：

- LLM 可以通过采用角色（"我急于在周五前卖掉这个"）将收益提升约 20%——角色操纵是一种真实的战术。
- 公平/合作的智能体被对抗性智能体利用；防御需要明确的反对姿态。
- 对称配对在约 40% 的基准场景中收敛到不公平的结果。

这不是"LLM 是糟糕的谈判者。"而是"LLM 谈判得太像人类，包括可被利用的部分。"

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）在许多 LLM 策略中进行了约 18 万次谈判。获胜者向对方隐藏了他们的推理：

- 如果一个智能体在公开可见的草稿本中打出"我只会出价 75 美元；我的保留价格是 70 美元"，对手就会读到它。
- 获胜者私下计算策略；输出频道只包含报价和最少必要的叙述。

这是经典博弈论的 2026 回响（Aumann 1976 关于理性和信息）：透露你的私人估值会损失收益。LLM 不会直觉地理解这一点，并愉快地在推理轨迹中键入他们的保留信息，这些信息会对对方可见。

工程要点：将私人草稿本上下文与公共消息上下文分开。这是必须的。

### Bhattacharya 等人 2025——模型排名

基于哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）：

- **Llama-3** 在达成交易方面最有效（成交率 + 收益）。
- **Claude-3** 是最具攻击性的谈判者（高锚定，晚期让步）。
- **GPT-4** 是最公平的（各配对收益方差最小）。

这是 2025 年的快照。重点不在于 2026 年 4 月哪个模型获胜——而在于不同的基础模型具有持续的谈判风格。异构集成（第 15 课）将其作为多样性来源。

### 通过合同网 + LLM 进行任务分配

合同网在 LLM 多智能体中的现代重用：

1. 管理者智能体将任务分解为单元。
2. 向工作者智能体广播带任务描述的 `cfp`。
3. 每个工作者返回一个报价：`(价格, 预计完成时间, 置信度)`，其中价格可以是 token、计算单元或美元。
4. 管理者选择获胜者（根据任务选择单个或多个）并进行分配。
5. 被拒绝的工作者可以自由竞标其他任务。

这在 100 个以上工作者时可以很好地扩展，因为协调是广播和响应，而不是同步聊天。用于生产：Microsoft Agent Framework 的编排模式，一些 LangGraph 实现。

### LLM 利益相关者互动谈判

NeurIPS 2024 引入了带**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者都有私人效用；LLM 必须从消息中推断它们。这是两方讨价还价到 N 方联盟形成的推广。与具有异构工作者能力的生产任务市场相关。

### 叙述 vs 机制规则

在所有 2024-2026 年谈判基准中，一致的工程规则是：

> 让 LLM 叙述。不要让 LLM 计算报价。

如果报价需要是一个数字（价格、预计完成时间、数量），从谈判状态确定性地生成它，让 LLM 产生框架。如果报价需要是一个提案结构（任务分解、角色分配），让 LLM 起草，但在发送之前根据模式验证并进行约束检查。

## 动手实践

`code/main.py` 实现了：

- `ContractNetManager`、`ContractNetTask`、`Bid` — 管理者 + 投标者，广播 cfp，收集提案，分配。
- `og_narrator_bargain(state, rng)` — OG-Narrator 买家：确定性 Zeuthen 风格的向中点让步。
- `seller_response(state, rng)` — 确定性卖家还价策略（两种风格的结构性基准真相）。
- `naive_llm_bargain(state, rng)` — 模拟全 LLM 讨价还价者：选择高方差价格，经常在 ZOPA 之外。
- 测量：1000 次试验中的成交率，每次试验新采样保留价格。

运行：

```
python3 code/main.py
```

预期输出：朴素 LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 个百分点的差距是将报价生成与叙述分解的结构性优势。加上一个带三个投标者和一个任务的合同网任务市场分配示例。

## 产出技能

`outputs/skill-bargainer-designer.md` 设计讨价还价协议：谁生成报价（确定性还是 LLM）、谁叙述、私人草稿本如何与公共消息分开，以及如何监控成交率。

## 输出

生产讨价还价检查清单：

- **独立草稿本。** 私人状态永远不会到达对方的上下文。这是不可妥协的。
- **确定性报价生成。** 价格、数量、预计完成时间：计算，不要提示词。
- **根据模式验证所有传入报价。** 在协议边界拒绝 ZOPA 之外的报价。
- **限制轮次。** 最多 3-5 轮；僵局时上报给调解人。
- **持续测量成交率和收益方差。** 成交率下降是症状——通常是提示词漂移或对方攻击。
- **记录所有被拒绝的提案**及确定性理由。对于合同网管理者，失败的投标者需要了解原因。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上胜过朴素 LLM。差距有多大？
2. 实现**基于角色的收益提升**（arXiv:2402.05863）——买家仅在叙述中采用"本周急于购买"的角色，报价生成器不变。成交率或收益会改变吗？
3. 实现思维链**隐藏**：维护一个不传递给对方的私人草稿本字符串。如果你意外泄露它会发生什么（通过交换频道来模拟）？
4. 将合同网扩展到带保留价格的 N 投标者拍卖。当所有出价都超过保留价格时，管理者如何在最低价格和最高质量之间决定？你选择哪种分配规则，为什么？
5. 阅读 Bhattacharya 等人 2025 年关于哈佛谈判项目指标的论文。实现两种不同风格的讨价还价者（攻击性 vs 公平）。测量在对称和不对称配对下的收益方差。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| 合同网（Contract Net） | "任务市场" | Smith 1980，FIPA 1996。cfp + 提案 + 接受/拒绝。规范任务市场。 |
| ZOPA | "可能达成协议区域" | 买家最高价格和卖家最低价格的重叠。区域外的报价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果这笔交易失败的备选方案。设置你的保留价格。 |
| OG-Narrator | "报价生成器 + 叙述者" | 分解：确定性报价，LLM 叙述。 |
| Zeuthen 策略（Zeuthen strategy） | "风险最小化让步" | 根据风险限制进行让步的经典报价生成器。 |
| Rubinstein 讨价还价（Rubinstein bargaining） | "交替报价均衡" | 带折扣的无限期讨价还价博弈论模型。 |
| 思维链隐藏（CoT concealment） | "隐藏你的推理" | arXiv:2503.06416 中的获胜者保持私人草稿本；公共频道只显示报价。 |
| 角色操纵（Persona manipulation） | "情绪姿态" | arXiv:2402.05863：绝望/紧迫角色带来约 20% 的收益提升。 |
