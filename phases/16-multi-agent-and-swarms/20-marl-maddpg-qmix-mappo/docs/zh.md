# MARL——MADDPG、QMIX、MAPPO

> 多智能体协调的强化学习遗产，在 2026 年仍然指导着 LLM 智能体系统。**MADDPG**（Lowe 等人，NeurIPS 2017，arXiv:1706.02275）引入了集中训练、去中心化执行（CTDE）：每个批评者在训练期间看到所有智能体的状态和行动；在测试时只有局部演员运行。适用于合作、竞争和混合设置。**QMIX**（Rashid 等人，ICML 2018，arXiv:1803.11485）是带单调混合网络的价值分解；每个智能体的 Q 值组合成联合 Q，使 `argmax` 能够干净地分布——在星际争霸多智能体挑战（SMAC）上占主导地位。**MAPPO**（Yu 等人，NeurIPS 2022，arXiv:2103.01955）是带集中价值函数的 PPO；在粒子世界、SMAC、Google Research Football、Hanabi 上以最少的调整就"出奇有效"。这些支撑了必须去中心化行动的智能体团队的训练策略。MAPPO 是 **2026 年默认的合作 MARL 基线**。本课在一个小型网格世界玩具上构建每种算法，并在接触 LLM 智能体训练之前将三个思想内化。

**类型：** 学习
**编程语言：** Python（标准库，小型无 NumPy 实现）
**前置知识：** Phase 09（强化学习）、Phase 16 · 09（并行群体网络）
**预计时间：** 约 90 分钟

## 问题背景

LLM 智能体系统越来越多地训练智能体间协调的策略：何时推迟，何时行动，调用哪个对等体。告诉你如何训练这类策略的文献是多智能体强化学习（MARL），它早于 LLM 浪潮，并有一小套主流算法。

在没有模式词汇的情况下阅读 MARL 论文是痛苦的。集中训练与去中心化执行（CTDE）、价值分解和集中批评者不是流行语——它们是针对具体问题的具体答案：

- 独立 RL（每个智能体单独学习）从每个智能体的角度来看是非平稳的。很差。
- 集中 RL（一个智能体控制所有）不能扩展并且违反执行约束。
- CTDE 两全其美：用全局信息训练，用局部策略部署。

## 核心概念

### 论文使用的三种环境

- **粒子世界（多智能体粒子环境）。** 带合作/竞争任务的简单 2D 物理。MADDPG 的原始测试床。
- **星际争霸多智能体挑战（SMAC）。** 合作微观管理，部分观察。QMIX 的测试床。离散行动，连续状态。
- **Google Research Football、Hanabi、MPE。** MAPPO 基线。

不同的环境有不同的行动/观察类型。算法相应地选择。

### MADDPG（2017）——CTDE 模式

每个智能体 `i` 有一个演员 `mu_i(o_i)`，将其自身观察映射到行动。每个智能体还有一个批评者 `Q_i(x, a_1, ..., a_n)`，在训练期间看到所有观察和所有行动。演员通过对批评者评估的策略梯度更新。

```
演员更新：    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) 在 a_i=mu_i(o_i) 时]
批评者更新：  Q_i(x, a_1..n) 的 TD，给定下一状态联合估计
```

为什么要 CTDE：在训练时，我们知道每个人的行动；我们用这来降低每个批评者的方差。在部署时，每个智能体只看到 `o_i` 并调用 `mu_i(o_i)`。

失败模式：批评者随 N 个智能体增长（输入包括所有行动）。没有近似，在约 10 个智能体之后无法扩展。

### QMIX（2018）——价值分解

仅合作。全局奖励是每个智能体 Q 值的单调函数的和：

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

单调性保证了 `argmax_a Q_tot` 可以由每个智能体独立选择 `argmax_{a_i} Q_i` 来计算。这**正是**你需要的去中心化执行属性。在训练时，混合网络从每个智能体的 Q 值生成 `Q_tot`。

为什么 QMIX 在 SMAC 上获胜：合作星际争霸微观管理有同质智能体、局部观察、全局奖励——完美适合价值分解。

失败模式：单调性约束是限制性的；有些任务的奖励结构不是单调可分解的（一个智能体为团队牺牲）。扩展（QTRAN、QPLEX）放宽了这个约束。

### MAPPO（2022）——被忽视的默认值

多智能体 PPO：带集中价值函数的 PPO。每个智能体有自己的策略；所有智能体共享（或有每智能体的）看到完整状态的价值函数。Yu 等人 2022 年在五个基准上对 MAPPO 与 MADDPG、QMIX 及其扩展进行了基准测试，发现：

- MAPPO 在粒子世界、SMAC、Google Research Football、Hanabi、MPE 上匹配或超越了离策略 MARL 方法。
- 所需超参数调整最少。
- 训练稳定；跨种子可复现。

社区低估了在策略 MARL，直到这篇论文。在 2026 年，MAPPO 是合作 MARL 的默认基线；任何新方法都必须超越它。

### 为什么 LLM 智能体工程师应该关心

三个直接用途：

1. **路由器训练。** 元智能体选择哪个子智能体处理任务。这是一个带 N 个去中心化子智能体和一个集中路由器的 MARL 问题。MAPPO 适合。
2. **角色涌现。** 在生成式智能体仿真中，训练智能体随时间采用互补角色是一个伪装的 MARL 问题。QMIX 风格的价值分解通过构造强制互补性。
3. **多智能体工具使用。** 当智能体共享工具并竞争预算时，通过 CTDE 训练它们会产生尊重资源约束的可部署局部策略。

实际说明：在 2026 年，大多数生产 LLM 智能体系统通过提示词来设置策略，而不是训练它们。当你有（a）大量交互数据、（b）明确的奖励信号，以及（c）投资训练基础设施的意愿时，MARL 才会发挥作用。

### CTDE 作为超越 RL 的设计模式

即使没有训练，CTDE 也是一个有用的架构模式：

- 在*设计*时，假设完整的团队可见性。
- 在*运行时*，强制去中心化执行：每个智能体只看到 `o_i`。

该模式迫使你明确保持每个智能体的状态，并从一开始就考虑部分可观察性。许多生产多智能体系统默默地假设处处都有共享状态——CTDE 纪律防止了这种情况。

### 非平稳性问题

当多个智能体同时学习时，每个智能体的环境（包括其他人的策略）是非平稳的。经典的单智能体 RL 证明不再成立。本课的 MARL 算法都解决了这个问题：

- MADDPG：全局批评者看到所有行动，因此其价值估计是平稳的。
- QMIX：价值分解将学习移动到定义了最优性的联合 Q 空间。
- MAPPO：集中价值函数减少了来自其他人策略变化的方差。

在 LLM 智能体系统中，非平稳性表现为"我的智能体上个月工作得很好，现在上游另一个智能体改变了，我的智能体表现异常"。用 CTDE 训练 MARL 是原则性的解决方案；提示词级别的修复更快但不那么持久。

### 本课不涵盖的内容

训练实际网络是第 09 阶段的主题。本课构建脚本化策略版本，在不更新梯度的情况下演示 CTDE、价值分解和集中价值模式。目标是在你拿起完整的 MARL 库（PyMARL、MARLlib、RLlib 多智能体）之前将这些模式内化。

## 动手实践

`code/main.py` 在一个微小的 2 智能体合作网格世界上实现了三个模式演示：

- 环境：4x4 网格上的 2 个智能体，一个奖励小球。奖励 = 1 如果任何智能体到达小球；任务完成。
- `IndependentAgents` — 每个智能体将其他智能体视为环境。基线。
- `MADDPGStyle` — 集中批评者计算联合价值；演员策略据此更新。脚本化策略改进。
- `QMIXStyle` — 带单调混合器的价值分解。
- `MAPPOStyle` — 集中价值函数；策略对共享基线进行更新。

所有四个运行相同的情节并报告平均到目标的步数。CTDE 变体收敛到比独立基线更短的路径。

运行：

```
python3 code/main.py
```

预期输出：独立智能体平均约 6 步；CTDE 变体收敛到约 3.5 步（4x4 网格的最优是 3 步）。尽管策略是脚本化的，模式差异仍然显现。

## 产出技能

`outputs/skill-marl-picker.md` 是一个为给定多智能体任务选择 MARL 算法的技能：合作 vs 竞争，同质 vs 异质，行动空间类型，规模，奖励信号。

## 输出

生产中的 MARL 很少见。当你确实使用时：

- **从 MAPPO 开始。** 2022 年的论文将其建立为基线；首先复现它可以节省几周追逐更复杂方法的时间。
- **记录每个智能体的观察和行动流。** 没有每个智能体的追踪，调试 MARL 是无望的。
- **将训练代码与执行代码分开。** CTDE 是一种纪律；让执行路径真正只看到 `o_i`。
- **奖励塑形警告。** MARL 对奖励设计极其敏感。塑形中一个协调错误，智能体就会学会利用它。运行对抗性测试。
- **对于 LLM 智能体**，首先考虑提示词级别的策略。只有在交互数据 + 奖励信号 + 基础设施都具备时才投资于 MARL 训练。

2. 实现一个竞争变体：两个智能体，一个小球，只有先到的才获得奖励。哪种模式能干净地处理竞争？历史上是 MADDPG。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| MARL | "多智能体 RL" | 多智能体系统的强化学习。 |
| CTDE | "集中训练，去中心化执行" | 用全局信息训练；用局部策略部署。 |
| MADDPG | "多智能体 DDPG" | CTDE，带看到所有观察 + 行动的每智能体批评者。 |
| QMIX | "价值分解" | 每智能体 Q 值的单调混合。合作。 |
| MAPPO | "多智能体 PPO" | 带集中价值函数的 PPO。2026 年默认基线。 |
| 价值分解（Value decomposition） | "个体 Q 的总和" | 联合 Q 表示为每智能体 Q 的单调函数。 |
| 非平稳性（Non-stationarity） | "移动目标" | 每个智能体的环境随其他人学习而改变。核心 MARL 问题。 |
| 在策略/离策略（On-policy / off-policy） | "从当前/回放学习" | PPO 是在策略（MAPPO）；DDPG 和 Q 学习是离策略。 |
| SMAC | "星际争霸多智能体挑战" | 合作微观管理基准；QMIX 的主战场。 |
